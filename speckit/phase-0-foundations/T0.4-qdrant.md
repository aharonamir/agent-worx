# T0.4 — Qdrant Instance

## Dependencies
None — foundational. Required before T2.3 (certified KB retrieval) and T2.7 (delta promotion
writes to certified KB).

## Files to Create
- `src/infra/qdrant_client.py`

## What to Build

A Qdrant collection `certified_kb` that stores the certified knowledge base — embeddings of
verified facts with metadata for filtering by agent type, phase, and confidence.

### Vector Configuration

| Setting | Value | Rationale |
|---|---|---|
| Vector size | 1536 | Matches `text-embedding-3-small` (set via `EMBEDDING_DIM` env var) |
| Distance | Cosine | Standard for normalized text embeddings |
| Payload indexes | `agent_type`, `phase`, `confidence`, `version` | All four are used in retrieval filters (T2.3) |

```python
# src/infra/qdrant_client.py

import os
from qdrant_client import QdrantClient
from qdrant_client.models import (
    Distance, VectorParams, PayloadSchemaType,
    PointStruct, Filter, FieldCondition, MatchValue, Range
)

QDRANT_URL = os.getenv("QDRANT_URL", "http://localhost:6333")
COLLECTION_NAME = os.getenv("QDRANT_COLLECTION", "certified_kb")
EMBEDDING_DIM = int(os.getenv("EMBEDDING_DIM", "1536"))

_client: QdrantClient | None = None


def get_client() -> QdrantClient:
    global _client
    if _client is None:
        _client = QdrantClient(url=QDRANT_URL)
    return _client


def setup_collection() -> None:
    """
    Create the certified_kb collection with payload indexes.
    Idempotent — safe to call on every startup.
    """
    client = get_client()
    existing = [c.name for c in client.get_collections().collections]

    if COLLECTION_NAME not in existing:
        client.create_collection(
            collection_name=COLLECTION_NAME,
            vectors_config=VectorParams(size=EMBEDDING_DIM, distance=Distance.COSINE),
        )

    # Payload indexes — idempotent, Qdrant ignores if already exists
    client.create_payload_index(COLLECTION_NAME, "agent_type", PayloadSchemaType.KEYWORD)
    client.create_payload_index(COLLECTION_NAME, "phase", PayloadSchemaType.KEYWORD)
    client.create_payload_index(COLLECTION_NAME, "confidence", PayloadSchemaType.FLOAT)
    client.create_payload_index(COLLECTION_NAME, "version", PayloadSchemaType.INTEGER)


def upsert_knowledge(point_id: str, vector: list[float], payload: dict) -> None:
    """
    Insert or update a certified knowledge entry.

    payload must include: agent_type, phase, confidence, version, text
    """
    client = get_client()
    client.upsert(
        collection_name=COLLECTION_NAME,
        points=[PointStruct(id=point_id, vector=vector, payload=payload)],
    )


def search_knowledge(
    query_vector: list[float],
    agent_type: str,
    min_confidence: float = 0.7,
    phases: list[str] | None = None,
    top_k: int = 5,
) -> list[dict]:
    """
    Search certified KB filtered by agent_type, confidence floor, and phase.
    Returns list of {id, score, payload}.
    """
    client = get_client()

    must_conditions = [
        FieldCondition(key="agent_type", match=MatchValue(value=agent_type)),
        FieldCondition(key="confidence", range=Range(gte=min_confidence)),
    ]
    if phases:
        # Qdrant doesn't support "IN" directly on keyword without a should-clause;
        # use should for OR semantics across phases
        should_conditions = [
            FieldCondition(key="phase", match=MatchValue(value=p)) for p in phases
        ]
        query_filter = Filter(must=must_conditions, should=should_conditions, min_should=1)
    else:
        query_filter = Filter(must=must_conditions)

    results = client.search(
        collection_name=COLLECTION_NAME,
        query_vector=query_vector,
        query_filter=query_filter,
        limit=top_k,
    )
    return [{"id": r.id, "score": r.score, "payload": r.payload} for r in results]


if __name__ == "__main__":
    import sys
    if len(sys.argv) > 1 and sys.argv[1] == "setup":
        setup_collection()
        print(f"Collection '{COLLECTION_NAME}' ready with payload indexes.")
```

## Acceptance Criteria

1. `docker compose up -d qdrant` brings the container to `healthy` within 15 seconds (verified via `/healthz`)
2. `python -m src.infra.qdrant_client setup` creates the `certified_kb` collection with vector size 1536, distance Cosine
3. All four payload indexes (`agent_type`, `phase`, `confidence`, `version`) exist on the collection
4. `upsert_knowledge()` followed by `search_knowledge()` with matching `agent_type` and `min_confidence` returns the upserted point
5. `search_knowledge()` with `min_confidence` higher than the point's confidence returns empty results
6. `search_knowledge()` with `phases=["journeyman", "work"]` returns points with `phase` in either value but not `apprentice`
7. Re-running `setup_collection()` on an existing collection does not error or duplicate indexes

```python
# tests/phase_0/test_infra_connectivity.py (additions)

import pytest
import uuid


@pytest.fixture
def qdrant(monkeypatch):
    monkeypatch.setenv("QDRANT_URL", "http://localhost:6333")
    monkeypatch.setenv("QDRANT_COLLECTION", "test_certified_kb")
    monkeypatch.setenv("EMBEDDING_DIM", "8")  # small dim for fast tests
    from src.infra import qdrant_client
    qdrant_client._client = None  # reset singleton for test isolation
    qdrant_client.setup_collection()
    yield qdrant_client
    qdrant_client.get_client().delete_collection("test_certified_kb")


def _fake_vector(seed: float, dim: int = 8) -> list[float]:
    return [seed] * dim


def test_qdrant_healthz():
    import httpx
    resp = httpx.get("http://localhost:6333/healthz")
    assert resp.status_code == 200


def test_collection_created_with_correct_config(qdrant):
    info = qdrant.get_client().get_collection("test_certified_kb")
    assert info.config.params.vectors.size == 8
    assert info.config.params.vectors.distance.name == "COSINE"


def test_upsert_and_search_roundtrip(qdrant):
    point_id = str(uuid.uuid4())
    qdrant.upsert_knowledge(
        point_id=point_id,
        vector=_fake_vector(0.9),
        payload={"agent_type": "burger-order", "phase": "journeyman", "confidence": 0.85, "version": 1, "text": "Cheeseburgers include a slice of cheddar by default"},
    )
    results = qdrant.search_knowledge(
        query_vector=_fake_vector(0.9),
        agent_type="burger-order",
        min_confidence=0.7,
    )
    assert len(results) == 1
    assert results[0]["id"] == point_id


def test_confidence_filter_excludes_low_confidence(qdrant):
    point_id = str(uuid.uuid4())
    qdrant.upsert_knowledge(
        point_id=point_id,
        vector=_fake_vector(0.9),
        payload={"agent_type": "burger-order", "phase": "journeyman", "confidence": 0.5, "version": 1, "text": "low confidence fact"},
    )
    results = qdrant.search_knowledge(
        query_vector=_fake_vector(0.9),
        agent_type="burger-order",
        min_confidence=0.7,
    )
    assert len(results) == 0


def test_phase_filter(qdrant):
    p_apprentice = str(uuid.uuid4())
    p_work = str(uuid.uuid4())
    qdrant.upsert_knowledge(p_apprentice, _fake_vector(0.9),
        {"agent_type": "burger-order", "phase": "apprentice", "confidence": 0.9, "version": 1, "text": "apprentice fact"})
    qdrant.upsert_knowledge(p_work, _fake_vector(0.9),
        {"agent_type": "burger-order", "phase": "work", "confidence": 0.9, "version": 1, "text": "work fact"})

    results = qdrant.search_knowledge(
        query_vector=_fake_vector(0.9),
        agent_type="burger-order",
        min_confidence=0.5,
        phases=["journeyman", "work"],
    )
    result_ids = {r["id"] for r in results}
    assert p_work in result_ids
    assert p_apprentice not in result_ids


def test_setup_collection_idempotent(qdrant):
    # Calling setup again must not raise
    qdrant.setup_collection()
    qdrant.setup_collection()
```

## Notes

- **`EMBEDDING_DIM=1536`** matches OpenAI's `text-embedding-3-small`. If you switch embedding
  models (e.g. to a local model with a different dimension), update both `EMBEDDING_DIM` and
  recreate the collection — Qdrant collections have a fixed vector size.
- **Payload index on `confidence` as FLOAT** enables range queries (`gte=min_confidence`),
  which is the core mechanism behind the work-phase auto-promotion threshold (Invariant I8)
  and the certified KB retrieval filter (T2.3).
- **The `should` + `min_should=1` pattern for phase filtering** is Qdrant's way of expressing
  "phase IN (journeyman, work)" — there's no native `IN` operator for a single field with OR
  semantics in a `must` clause, so `should` with `min_should=1` is the idiomatic equivalent.
- **Run `setup_collection()` from the Temporal worker's startup too**, not just the API —
  both processes may start before the other, and the collection must exist before either
  attempts to read/write.
