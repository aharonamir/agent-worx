# T0.3 — Redis Instance

## Dependencies
None — foundational. Required before T0.8 (Redis key schema), T1.1 (operational state),
T1.6 (readiness scorer), and T3.6 (circuit breaker).

## Files to Create
- `src/infra/redis_client.py`

## What to Build

A thin Redis client wrapper providing the four primitive operations used throughout the
framework: simple KV (`get`/`set`), hash operations (for circuit breaker stats), and
Streams (for the per-workflow context bus).

```python
# src/infra/redis_client.py

import os
import redis

REDIS_URL = os.getenv("REDIS_URL", "redis://localhost:6379/0")

_client: redis.Redis | None = None


def get_client() -> redis.Redis:
    global _client
    if _client is None:
        _client = redis.from_url(REDIS_URL, decode_responses=True)
    return _client


# ── Simple KV ────────────────────────────────────────────────────────────────

def redis_get(key: str) -> str | None:
    return get_client().get(key)


def redis_set(key: str, value: str, ttl_seconds: int | None = None) -> None:
    if ttl_seconds:
        get_client().set(key, value, ex=ttl_seconds)
    else:
        get_client().set(key, value)


def redis_delete(key: str) -> None:
    get_client().delete(key)


# ── Hash operations (circuit breaker stats, etc.) ─────────────────────────────

def redis_hset(key: str, mapping: dict[str, str]) -> None:
    get_client().hset(key, mapping=mapping)


def redis_hgetall(key: str) -> dict[str, str]:
    return get_client().hgetall(key)


def redis_hget(key: str, field: str) -> str | None:
    return get_client().hget(key, field)


# ── Streams (per-workflow context bus) ─────────────────────────────────────────

def stream_add(stream_key: str, fields: dict[str, str], maxlen: int = 1000) -> str:
    """Add an entry to a stream. Returns the entry ID."""
    return get_client().xadd(stream_key, fields, maxlen=maxlen, approximate=True)


def stream_read(stream_key: str, last_id: str = "0", count: int = 10) -> list:
    """Read entries from a stream after last_id."""
    result = get_client().xread({stream_key: last_id}, count=count, block=100)
    return result


def stream_set_ttl(stream_key: str, ttl_seconds: int) -> None:
    """Set expiry on a workflow-scoped stream — used to auto-clean on workflow close."""
    get_client().expire(stream_key, ttl_seconds)
```

### Redis-Stack Module Verification

The `redis/redis-stack` image bundles RediSearch, RedisJSON, and other modules.
Verify RediSearch is active (used for hot certified-KB query caching in T2.3):

```python
def verify_redisearch_module() -> bool:
    """Check RediSearch module is loaded."""
    modules = get_client().execute_command("MODULE", "LIST")
    module_names = [m[1] for m in modules]  # each entry: [b'name', b'search', b'ver', N, ...]
    return any(b"search" in str(name).lower().encode() if isinstance(name, str) else name == b"search"
               for name in module_names)
```

## Acceptance Criteria

1. `docker compose up -d redis` brings the container to `healthy` within 10 seconds
2. `redis-cli PING` returns `PONG`
3. `redis_set("test:key", "value")` followed by `redis_get("test:key")` returns `"value"`
4. RediSearch module is loaded (verified via `MODULE LIST`)
5. `stream_add()` + `stream_read()` round-trip returns the same fields that were written
6. `maxlen=1000` on `stream_add` caps stream growth — writing 1500 entries leaves approximately 1000
7. Persistence is enabled — restarting the container preserves previously-set keys (AOF + RDB)

```python
    assert redis_hget("circuit:test-agent:stats", "consecutive_failures") == "2"

    from src.infra.redis_client import verify_redisearch_module
    assert verify_redisearch_module() is True


def test_stream_roundtrip(redis_client):
    from src.infra.redis_client import stream_add, stream_read
    stream_add("stream:workflow:test-1:context", {"event": "task_started", "task_id": "t1"})
    result = stream_read("stream:workflow:test-1:context", last_id="0")
    assert len(result) == 1
    stream_name, entries = result[0]
    assert len(entries) == 1
    entry_id, fields = entries[0]
    assert fields["event"] == "task_started"


def test_stream_maxlen_caps_growth(redis_client):
    from src.infra.redis_client import stream_add, get_client
    for i in range(1500):
        stream_add("stream:workflow:test-2:context", {"event": f"e{i}"}, maxlen=1000)
    length = get_client().xlen("stream:workflow:test-2:context")
    # approximate=True means it won't be exactly 1000, but should be close
    assert length <= 1100
    assert length >= 900


def test_stream_ttl(redis_client):
    from src.infra.redis_client import stream_add, stream_set_ttl, get_client
    stream_add("stream:workflow:test-3:context", {"event": "started"})
    stream_set_ttl("stream:workflow:test-3:context", 3600)
    ttl = get_client().ttl("stream:workflow:test-3:context")
    assert 0 < ttl <= 3600
```

## Notes

- **Use `redis/redis-stack` image, not plain `redis`.** The plain image does not include
  RediSearch, which T2.3 (certified KB cache) and future vector caching depend on.
  See `docker-compose.yml` — `redis` service uses `redis-stack:7.2.0-v9`.
- **`decode_responses=True`** is set globally on the client — all string operations return
  `str`, not `bytes`. This matches the type hints used throughout `redis_client.py` and
  downstream consumers (T1.6, T3.6).
- **Stream key naming convention:** `stream:workflow:{workflow_id}:context` — established
  here, used by T2.x context bus implementation. The `stream_set_ttl` helper should be called
  by the orchestrator when a workflow closes (24h TTL per `shared/PROJECT_STRUCTURE.md`).
- **`maxlen` with `approximate=True`** trades exactness for performance — Redis trims in
  batches rather than on every write. This is the correct choice for high-throughput streams;
  exact trimming (`approximate=False`) would add latency to every `XADD`.
