## What to Build

This is the human gate between apprentice and journeyman phases. `GET /cert1-status`
surfaces an **advisory** signal -- the readiness score plus simulation violation status --
but the only thing that can change `agent:{name}:phase` is an explicit
`POST /cert1/approve` carrying a real `expert_id`.

### Advisory Signal Definition

```
cert_ready = readiness_score >= READINESS_CERT1_THRESHOLD (0.80)
             AND violations_unresolved == 0 (from the most recent simulation session)
```

Both `cert_ready` and `readiness_score` are surfaced to the expert but **never block**
`POST /cert1/approve`. Per Invariant I2, only the expert's action changes phase --
the system never auto-graduates an agent, and it also never refuses to graduate one
based on a number. (Contrast with T2.8's Cert 2 gate, which has no score gate at all --
Cert 1 at least has *advisory* numbers because the readiness score is the primary signal
this phase produces.)

```python
# src/apprentice/cert1_service.py

from datetime import datetime
from src.core.models import CertEvent, CertApproveRequest, CertRejectRequest
from src.core.enums import CertGate, CertDecision, Phase
from src.core.exceptions import AgentNotInPhase
from src.infra.redis_client import redis_get, set_agent_phase, get_agent_state
from src.infra.postgres_client import insert_cert_event, get_pool

JOURNEYMAN_LEARNING_RATE = 0.5


async def get_cert1_status(agent_type: str) -> dict:
    """
    Advisory signal only. Does not gate /cert1/approve.
    """
    state = get_agent_state(agent_type)
    readiness_score = state["readiness_score"]
    cert_ready_flag = state["cert_ready"]

    violations_unresolved = await _latest_simulation_unresolved_violations(agent_type)
    simulation_runs = await _count_simulation_sessions(agent_type)

    return {
        "readiness_score": readiness_score,
        "simulation_runs_completed": simulation_runs,
        "violations_unresolved": violations_unresolved,
        "cert_ready": cert_ready_flag and violations_unresolved == 0,
    }


async def approve_cert1(agent_type: str, request: CertApproveRequest) -> dict:
    """
    The ONLY function in the codebase that may transition an agent type from
    apprentice to journeyman. Invariant I2: requires a real expert_id, no
    automated caller path exists to this function other than the API route
    handling POST /cert1/approve.
    """
    state = get_agent_state(agent_type)
    if state["phase"] != Phase.APPRENTICE.value:
        raise AgentNotInPhase(agent_type, expected=Phase.APPRENTICE, actual=Phase(state["phase"]))

    # Record the cert event BEFORE flipping phase -- if phase update fails,
    # we want a record that approval was attempted at this readiness score.
    event = CertEvent(
        agent_type=agent_type,
        gate=CertGate.CERT1,
        decision=CertDecision.APPROVED,
        expert_id=request.expert_id,
        readiness_score=state["readiness_score"],
        notes=request.notes,
    )
    await insert_cert_event(event)

    # The ONLY place agent:{name}:phase transitions apprentice -> journeyman
    set_agent_phase(agent_type, Phase.JOURNEYMAN, JOURNEYMAN_LEARNING_RATE)

    return {
        "cert_event_id": event.id,
        "previous_phase": Phase.APPRENTICE.value,
        "new_phase": Phase.JOURNEYMAN.value,
        "new_learning_rate": JOURNEYMAN_LEARNING_RATE,
    }


async def reject_cert1(agent_type: str, request: CertRejectRequest) -> dict:
    """
    Records the rejection. Phase remains apprentice -- the agent continues
    Q&A sessions (T1.3/T1.4) to address the gap the expert identified.
    """
    state = get_agent_state(agent_type)
    if state["phase"] != Phase.APPRENTICE.value:
        raise AgentNotInPhase(agent_type, expected=Phase.APPRENTICE, actual=Phase(state["phase"]))

    event = CertEvent(
        agent_type=agent_type,
        gate=CertGate.CERT1,
        decision=CertDecision.REJECTED,
        expert_id=request.expert_id,
        readiness_score=state["readiness_score"],
        notes=request.reason,
    )
    await insert_cert_event(event)

    return {"cert_event_id": event.id, "phase": Phase.APPRENTICE.value}


async def _latest_simulation_unresolved_violations(agent_type: str) -> int:
    """
    Look up the most recently closed simulation session's violations_unresolved.
    If no session has been closed yet, treat as having unresolved violations
    (conservative default -- an agent that has never been simulated is not cert_ready).
    """
    pool = get_pool()
    async with pool.acquire() as conn:
        # Simulation session results aren't persisted in this spec-kit's current
        # schema (T1.9 sessions are in-memory) -- this function reads from a
        # lightweight summary table populated by T1.9's close_session.
        # If that table/row doesn't exist yet, return 1 (conservative: not ready).
        row = await conn.fetchrow("""
            SELECT violations_unresolved FROM simulation_session_summaries
            WHERE agent_type = $1 ORDER BY closed_at DESC LIMIT 1
        """, agent_type)

    if row is None:
        return 1  # no simulation run yet -> not cert_ready
    return row["violations_unresolved"]


async def _count_simulation_sessions(agent_type: str) -> int:
    pool = get_pool()
    async with pool.acquire() as conn:
        row = await conn.fetchrow("""
            SELECT COUNT(*) AS cnt FROM simulation_session_summaries WHERE agent_type = $1
        """, agent_type)
    return row["cnt"] if row else 0
```

### Custom Exception

```python
# Add to src/core/exceptions.py

class AgentNotInPhase(Exception):
    def __init__(self, agent_type: str, expected, actual):
        self.agent_type = agent_type
        self.expected = expected
        self.actual = actual
        super().__init__(
            f"Agent '{agent_type}' is in phase '{actual.value}', "
            f"but this operation requires phase '{expected.value}'"
        )
```

### API Route

```python
# src/api/routes/cert.py

from fastapi import APIRouter, HTTPException
from src.core.models import CertApproveRequest, CertRejectRequest
from src.core.exceptions import AgentNotInPhase
from src.apprentice.cert1_service import get_cert1_status, approve_cert1, reject_cert1
from src.core.agent_type_service import get_agent_type_by_name

router = APIRouter(prefix="/agent-types", tags=["cert"])


@router.get("/{agent_type_id}/cert1-status")
async def cert1_status(agent_type_id: str):
    agent = await get_agent_type_by_name(agent_type_id)
    return await get_cert1_status(agent.name)


@router.post("/{agent_type_id}/cert1/approve")
async def cert1_approve(agent_type_id: str, body: CertApproveRequest):
    agent = await get_agent_type_by_name(agent_type_id)
    try:
        return await approve_cert1(agent.name, body)
    except AgentNotInPhase as e:
        raise HTTPException(status_code=409, detail=str(e))


@router.post("/{agent_type_id}/cert1/reject")
async def cert1_reject(agent_type_id: str, body: CertRejectRequest):
    agent = await get_agent_type_by_name(agent_type_id)
    try:
        return await reject_cert1(agent.name, body)
    except AgentNotInPhase as e:
        raise HTTPException(status_code=409, detail=str(e))
```

## Acceptance Criteria

1. `GET /cert1-status` on an agent with `readiness_score=0.85` and zero simulation
   sessions returns `cert_ready=False` (no simulation evidence yet, conservative default)
2. `GET /cert1-status` on an agent with `readiness_score=0.85` and a closed simulation
   session with `violations_unresolved=0` returns `cert_ready=True`
3. `POST /cert1/approve` succeeds (200) even when `readiness_score < 0.80` -- the score
   is advisory only and never blocks approval
4. `POST /cert1/approve` on success: writes a `cert_events` row with
   `gate='cert1', decision='approved'`; sets `agent:{name}:phase=journeyman` and
   `agent:{name}:learning_rate=0.5` in Redis
5. `POST /cert1/approve` on an agent already in `journeyman` or `work` phase returns `409`
6. `POST /cert1/reject` writes a `cert_events` row with `decision='rejected'` and leaves
   `phase=apprentice` unchanged
7. No code path other than `approve_cert1` ever writes `Phase.JOURNEYMAN` to
   `agent:{name}:phase` (grep check, Invariant I2)

```python
    only appear in cert1_service.py (this task) and cert2_service.py (T2.8, the
    journeyman -> work transition, which is a different transition entirely).

    import re

    pattern = re.compile(r'set_agent_phase\([^)]*Phase\.JOURNEYMAN')
    allowed_files = {"cert1_service.py"}

    violations = []
    for path in pathlib.Path("src").rglob("*.py"):
        if path.name in allowed_files:
            continue
        text = path.read_text()
        if pattern.search(text):
            violations.append(str(path))

    assert violations == [], f"Unexpected Phase.JOURNEYMAN transition found in: {violations}"
```

## Notes

- **Why `cert_ready` requires BOTH a high readiness score AND a clean simulation run,
  but neither gates approval:** these are two independent pieces of evidence the system
  surfaces to help the expert decide -- the readiness score says "the knowledge graph is
  dense," the simulation result says "the agent coordinates correctly in practice." An
  expert might reasonably approve an agent with `readiness_score=0.72` if they've
  personally observed it handle edge cases well in simulation, or reject one at `0.95`
  if the simulation revealed a coordination problem the score doesn't capture. The
  combined `cert_ready` boolean is a convenience summary, never a gate.
- **The conservative default (`cert_ready=False` with no simulation session) matters**
  because T1.9's simulation sessions are optional from the system's perspective -- an
  agent type with `topology=pipeline` and a single agent (no multi-agent coordination at
  all) might never need a meaningful simulation. In practice, T1.7's proposal engine
  still produces a single-agent `ProposalArtifact`, and T1.9's `start_session` works
  even for single-agent compositions (the "simulation" is just the expert confirming the
  single agent's input/output contract makes sense). The expert can always override via
  `/cert1/approve` regardless of this default.
- **`simulation_session_summaries` is a new table introduced by this task**, not part of
  T0.7's original five. It exists because T1.9 keeps session state in-memory
  (process-local) but Cert 1's advisory signal needs to survive process restarts and be
  queryable independent of any specific in-memory session. T1.9's `close_session()` must
  be updated to also write to this table -- treat this as a cross-cutting dependency
  between T1.9 and T1.11 and implement both INSERT statements together.
- **The grep-based invariant test is deliberately narrow** (`set_agent_phase(...
  Phase.JOURNEYMAN`) -- it allows `cert1_service.py` as the sole writer of this specific
  transition. T2.8 (Cert 2) will have its own analogous test for the
  `journeyman -> work` transition, scoped to `cert2_service.py`. Together these two tests
  give Invariant I2 concrete, automated enforcement: phase transitions are
  grep-traceable to exactly one file each.
- **`notes` vs `reason`:** `CertApproveRequest.notes` is optional context the expert
  may add ("strong allergen coverage, approved"); `CertRejectRequest.reason` is required
  (min_length=10) because a rejection without a stated reason gives the apprentice loop
  (T1.2/T1.3) nothing actionable to target in the next Q&A round.
