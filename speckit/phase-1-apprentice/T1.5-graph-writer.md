# T1.5 — Graph Writer

## Dependencies
- T1.4 complete (`AnswerProcessorResult` with `status="write"` is the input)
- T0.6 complete (Kuzu schema with `Concept` nodes and `RELATES_TO` edges)
- Load `shared/DATA_SCHEMAS.md` — consumes `AnswerProcessorResult`, `ExtractedEntity`, `ExtractedRelation`

## Files to Create
- `src/apprentice/graph_writer.py`

## What to Build

Commits validated entities and relations to the Kuzu domain graph. Each write assigns a
confidence score based on how the knowledge was obtained — direct expert statements get
the highest confidence, inferred relations get less. After writing, this module triggers
the readiness scorer (T1.6) to keep the readiness score current.

### Confidence Assignment Rules

| Source of knowledge | Confidence assigned |
|---|---|
| Direct statement, high extraction confidence (≥ 0.85) | `0.9` |
| Direct statement, moderate extraction confidence (0.5–0.85) | `0.8` |
| Follow-up clarification (answer to a generated follow-up question) | `0.8` |
| Inferred relation (entity mentioned but relation type inferred, not stated) | `0.6` |

```python
# src/apprentice/graph_writer.py

import uuid
from datetime import datetime
from src.core.models import AnswerProcessorResult, ExtractedEntity, ExtractedRelation
from src.infra.kuzu_client import execute

DIRECT_HIGH_CONFIDENCE = 0.9
DIRECT_MODERATE_CONFIDENCE = 0.8
FOLLOWUP_CONFIDENCE = 0.8
INFERRED_CONFIDENCE = 0.6


def write_to_graph(agent_type: str, result: AnswerProcessorResult, is_followup: bool = False) -> dict:
    """
    Write validated entities and relations to the Kuzu domain graph.

    Precondition: result.status == "write" (caller's responsibility -- T1.4's route
    handler only calls this when status="write").

    Returns a summary dict: {nodes_created, nodes_matched, edges_created, edges_updated}
    """
    if result.status != "write":
        raise ValueError(f"write_to_graph called with status={result.status!r}, expected 'write'")

    now_str = datetime.utcnow().isoformat()
    summary = {"nodes_created": 0, "nodes_matched": 0, "edges_created": 0, "edges_updated": 0}

    # Step 1: Ensure all entities exist as Concept nodes (MERGE semantics)
    label_to_id: dict[str, str] = {}
    for entity in result.entities:
        concept_id = _find_or_create_concept(agent_type, entity, now_str, summary)
        label_to_id[entity.label] = concept_id

    # Step 2: Write/update relations
    for relation in result.relations:
        confidence = _assign_confidence(relation, is_followup)
        _upsert_relation(agent_type, relation, confidence, label_to_id, summary)

    return summary


def _find_or_create_concept(agent_type: str, entity: ExtractedEntity, now_str: str, summary: dict) -> str:
    """
    Find an existing Concept by label, or create a new one.
    Returns the concept's id.

    If the concept already exists (e.g. it was a root topic node from T1.1 with
    confidence=0.0), its confidence is updated to the entity's extraction confidence
    if that confidence is higher -- confidence only ever increases via this path.
    """
    existing = execute(agent_type,
        "MATCH (c:Concept {label: $label}) RETURN c.id AS id, c.confidence AS confidence",
        {"label": entity.label}
    ).get_as_df()

    if len(existing) > 0:
        concept_id = existing.iloc[0]["id"]
        current_confidence = float(existing.iloc[0]["confidence"])
        if entity.confidence > current_confidence:
            execute(agent_type,
                "MATCH (c:Concept {id: $id}) SET c.confidence = $confidence",
                {"id": concept_id, "confidence": entity.confidence}
            )
        summary["nodes_matched"] += 1
        return concept_id

    concept_id = f"concept-{uuid.uuid4().hex[:12]}"
    execute(agent_type, """
        CREATE (:Concept {
            id: $id, label: $label, namespace: 'task',
            confidence: $confidence, created_at: $created_at
        })
    """, {
        "id": concept_id, "label": entity.label,
        "confidence": entity.confidence, "created_at": now_str,
    })
    summary["nodes_created"] += 1
    return concept_id


def _assign_confidence(relation: ExtractedRelation, is_followup: bool) -> float:
    if is_followup:
        return FOLLOWUP_CONFIDENCE
    if relation.confidence >= 0.85:
        return DIRECT_HIGH_CONFIDENCE
    if relation.confidence >= 0.5:
        return DIRECT_MODERATE_CONFIDENCE
    return INFERRED_CONFIDENCE


def _upsert_relation(agent_type: str, relation: ExtractedRelation, confidence: float,
                      label_to_id: dict[str, str], summary: dict) -> None:
    """
    Create or update a RELATES_TO edge between two concepts.

    If an edge with the same relation_type already exists between source and target,
    update its confidence (taking the max of old and new) and bump version.
    Otherwise create a new edge at version=1.

    Concepts referenced by `relation.source`/`relation.target` may not be in
    `label_to_id` if they were already-existing concepts not newly extracted as
    entities in this turn (e.g. the answer referenced a concept by name without
    it appearing in the `entities` list). In that case, resolve by label lookup.
    """
    source_id = label_to_id.get(relation.source) or _resolve_concept_id(agent_type, relation.source)
    target_id = label_to_id.get(relation.target) or _resolve_concept_id(agent_type, relation.target)

    if source_id is None or target_id is None:
        # Cannot write an edge to a concept that doesn't exist and wasn't extracted
        # as an entity this turn -- skip silently. This is a rare edge case where
        # the LLM extraction included a relation referencing an unextracted entity.
        return

    existing = execute(agent_type, """
        MATCH (a:Concept {id: $source_id})-[r:RELATES_TO {relation: $relation_type}]->(b:Concept {id: $target_id})
        RETURN r.confidence AS confidence, r.version AS version
    """, {"source_id": source_id, "target_id": target_id, "relation_type": relation.relation_type}).get_as_df()

    if len(existing) > 0:
        old_confidence = float(existing.iloc[0]["confidence"])
        old_version = int(existing.iloc[0]["version"])
        new_confidence = max(old_confidence, confidence)
        execute(agent_type, """
            MATCH (a:Concept {id: $source_id})-[r:RELATES_TO {relation: $relation_type}]->(b:Concept {id: $target_id})
            SET r.confidence = $confidence, r.version = $version
        """, {
            "source_id": source_id, "target_id": target_id, "relation_type": relation.relation_type,
            "confidence": new_confidence, "version": old_version + 1,
        })
        summary["edges_updated"] += 1
    else:
        execute(agent_type, """
            MATCH (a:Concept {id: $source_id}), (b:Concept {id: $target_id})
            CREATE (a)-[:RELATES_TO {relation: $relation_type, confidence: $confidence, version: 1}]->(b)
        """, {
            "source_id": source_id, "target_id": target_id,
            "relation_type": relation.relation_type, "confidence": confidence,
        })
        summary["edges_created"] += 1


def _resolve_concept_id(agent_type: str, label: str) -> str | None:
    df = execute(agent_type,
        "MATCH (c:Concept {label: $label}) RETURN c.id AS id", {"label": label}
    ).get_as_df()
    return df.iloc[0]["id"] if len(df) > 0 else None
```

## Acceptance Criteria

1. Writing a result with two new entities and one relation creates 2 `Concept` nodes and 1 `RELATES_TO` edge
2. Writing an entity whose `label` matches an existing root concept (confidence=0.0 from T1.1) updates that node's confidence rather than creating a duplicate
3. Writing the same relation twice: first write creates the edge at the assigned confidence and `version=1`; second write updates confidence to `max(old, new)` and increments `version` to `2`
4. `is_followup=True` assigns `FOLLOWUP_CONFIDENCE` (0.8) regardless of the extraction's reported `relation.confidence`
5. `write_to_graph()` called with `status != "write"` raises `ValueError`
6. A relation referencing a `source`/`target` label that exists nowhere in the graph and wasn't in `entities` is skipped without error (summary counts remain unchanged for that relation)

```python
# tests/phase_1/test_graph_writer.py

import pytest
from src.apprentice.graph_writer import write_to_graph, FOLLOWUP_CONFIDENCE, DIRECT_HIGH_CONFIDENCE
from src.core.models import AnswerProcessorResult, ExtractedEntity, ExtractedRelation
from src.infra.kuzu_client import initialize_schema, execute


@pytest.fixture
def agent_type(tmp_path, monkeypatch):
    monkeypatch.setenv("KUZU_GRAPHS_DIR", str(tmp_path))
    atype = "test-graph-writer"
    initialize_schema(atype)
    return atype


def _write_result(entities, relations, confidence=0.9):
    return AnswerProcessorResult(
        status="write",
        entities=[ExtractedEntity(**e) for e in entities],
        relations=[ExtractedRelation(**r) for r in relations],
        confidence=confidence,
    )


def test_creates_nodes_and_edge(agent_type):
    result = _write_result(
        entities=[
            {"label": "cheeseburger", "entity_type": "menu_item", "confidence": 0.9},
            {"label": "cheddar cheese", "entity_type": "ingredient", "confidence": 0.9},
        ],
        relations=[
            {"source": "cheeseburger", "target": "cheddar cheese", "relation_type": "includes", "confidence": 0.9},
        ],
    )
    summary = write_to_graph(agent_type, result)

    assert summary["nodes_created"] == 2
    assert summary["edges_created"] == 1

    nodes = execute(agent_type, "MATCH (c:Concept) RETURN c.label AS label").get_as_df()
    assert set(nodes["label"]) == {"cheeseburger", "cheddar cheese"}

    edges = execute(agent_type,
        "MATCH (a:Concept)-[r:RELATES_TO]->(b:Concept) RETURN r.relation AS rel, r.confidence AS conf, r.version AS ver"
    ).get_as_df()
    assert edges.iloc[0]["rel"] == "includes"
    assert edges.iloc[0]["conf"] == DIRECT_HIGH_CONFIDENCE
    assert edges.iloc[0]["ver"] == 1


def test_updates_existing_root_concept_confidence(agent_type):
    # Root concept from T1.1 -- confidence=0.0
    execute(agent_type,
        "CREATE (:Concept {id:'root-1', label:'allergens', namespace:'task', confidence:0.0, created_at:'2025-01-01'})"
    )
    result = _write_result(
        entities=[{"label": "allergens", "entity_type": "topic", "confidence": 0.85}],
        relations=[],
    )
    summary = write_to_graph(agent_type, result)

    assert summary["nodes_created"] == 0
    assert summary["nodes_matched"] == 1

    nodes = execute(agent_type, "MATCH (c:Concept {label:'allergens'}) RETURN c.confidence AS conf").get_as_df()
    assert nodes.iloc[0]["conf"] == 0.85


def test_repeated_relation_increments_version_and_takes_max_confidence(agent_type):
    result1 = _write_result(
        entities=[
            {"label": "burger", "entity_type": "menu_item", "confidence": 0.6},
            {"label": "lettuce", "entity_type": "ingredient", "confidence": 0.6},
        ],
        relations=[
            {"source": "burger", "target": "lettuce", "relation_type": "includes", "confidence": 0.6},
        ],
    )
    write_to_graph(agent_type, result1)  # confidence 0.6 -> DIRECT_MODERATE (0.8)

    result2 = _write_result(
        entities=[
            {"label": "burger", "entity_type": "menu_item", "confidence": 0.95},
            {"label": "lettuce", "entity_type": "ingredient", "confidence": 0.95},
        ],
        relations=[
            {"source": "burger", "target": "lettuce", "relation_type": "includes", "confidence": 0.95},
        ],
    )
    summary2 = write_to_graph(agent_type, result2)  # confidence 0.95 -> DIRECT_HIGH (0.9)

    assert summary2["edges_updated"] == 1
    assert summary2["edges_created"] == 0

    edges = execute(agent_type,
        "MATCH (a:Concept{label:'burger'})-[r:RELATES_TO]->(b:Concept{label:'lettuce'}) "
        "RETURN r.confidence AS conf, r.version AS ver"
    ).get_as_df()
    assert edges.iloc[0]["conf"] == DIRECT_HIGH_CONFIDENCE
    assert edges.iloc[0]["ver"] == 2


def test_followup_confidence_overrides_extraction_confidence(agent_type):
    result = _write_result(
        entities=[
            {"label": "fries", "entity_type": "menu_item", "confidence": 0.95},
            {"label": "ketchup", "entity_type": "ingredient", "confidence": 0.95},
        ],
        relations=[
            {"source": "fries", "target": "ketchup", "relation_type": "served_with", "confidence": 0.95},
        ],

    write_to_graph(agent_type, result, is_followup=True)

    edges = execute(agent_type,
        "MATCH (a:Concept{label:'fries'})-[r:RELATES_TO]->(b:Concept{label:'ketchup'}) RETURN r.confidence AS conf"
    ).get_as_df()
    assert edges.iloc[0]["conf"] == FOLLOWUP_CONFIDENCE  # 0.8, not 0.9 despite extraction confidence 0.95


def test_raises_on_non_write_status(agent_type):

    result = AnswerProcessorResult(status="conflict", confidence=0.9)
    with pytest.raises(ValueError):
        write_to_graph(agent_type, result)


def test_relation_to_unresolvable_concept_skipped_silently(agent_type):

    assert summary["edges_updated"] == 0
```

## Notes

- **MERGE-by-label semantics:** Kuzu (at the version pinned in `shared/DEPENDENCIES.md`)
  does not support Cypher's native `MERGE` clause with the same guarantees as Neo4j, so
  `_find_or_create_concept` implements merge-by-label manually via a read-then-write
  pattern. This is safe under the single-writer-per-agent-type model (Invariant I10) —
  there is no concurrent writer that could create a race between the existence check
  and the create.
- **Confidence only increases, never decreases, for nodes.** A concept's confidence
  represents "how sure are we this concept is real and relevant" — once an expert
  confirms a concept directly, a later lower-confidence inferential mention should not
  walk that back. Edge confidence uses the same max-on-update rule for the same reason.
- **`version` increments on every edge update**, even if the confidence doesn't change
  numerically (e.g. `max(0.9, 0.9) = 0.9`). This gives T0.7/Phase 3's re-certification
  logic (Invariant tied to T3.8) a monotonically increasing counter to detect "this edge
  was touched again," independent of whether the confidence value itself moved.
- **Silent skip on unresolvable relations** is a deliberate choice over raising an error.
  LLM extraction occasionally produces a relation that references a entity it didn't
  also list in `entities` (a minor extraction inconsistency). Failing the entire answer
  write over one malformed relation would discard otherwise-valid entities and relations
  from the same answer. The summary dict lets callers detect this happened (edges_created
  + edges_updated < len(relations)) if they want to log it.
- **This module does NOT call the readiness scorer.** Per T1.4's route handler design,
  `compute_and_store()` (T1.6) is called by the API route *after* `write_to_graph()`
  returns — keeping T1.5 focused purely on graph mutation, with scoring as an explicit
  separate step the orchestrating route controls.
