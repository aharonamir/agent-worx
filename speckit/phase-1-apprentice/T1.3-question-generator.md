# T1.3 — Socratic Question Generator

## Dependencies
- T1.2 complete (`detect_gaps()` returns `GapDetectorResult`)
- Load `shared/DATA_SCHEMAS.md` — uses `KnowledgeGap`, `QuestionGeneratorResult`
- Anthropic API client configured (`ANTHROPIC_API_KEY`)

## Files to Create
- `src/apprentice/question_generator.py`
- Add `POST /agent-types/{agent_type_id}/qa/question` to `src/api/routes/apprentice.py`

## What to Build

Takes the top-ranked gap from T1.2's output and generates exactly one focused question
for the domain expert. The generator must respect two hard constraints: never repeat a
concept the graph already knows with high confidence, and never ask outside `task_boundary`.

```python
# src/apprentice/question_generator.py

import json
from anthropic import Anthropic
from src.core.models import KnowledgeGap, QuestionGeneratorResult, GapDetectorResult
from src.apprentice.gap_detector import detect_gaps
from src.infra.kuzu_client import execute
from src.infra.postgres_client import get_pool

_client = Anthropic()

QUESTION_PROMPT = """You are the learning module for a specialized AI agent being trained \
in apprentice phase. Your job is to ask a domain expert ONE precise question that will \
resolve a specific gap in your knowledge graph.

AGENT DOMAIN:
  Goal: {goal}
  Task boundary: {task_boundary}

KNOWLEDGE GAP TO RESOLVE:
  Concept: "{concept_label}"
  Gap type: {gap_type}
  Namespace: {namespace}

WHAT YOU ALREADY KNOW (do not ask about these — already covered with high confidence):
{known_concepts}

RULES:
- Ask exactly ONE question.
- The question must stay strictly within the task boundary above.
- The question must target the specific gap — not a generic "tell me about X".
- If gap_type is "orphan", ask how this concept relates to other concepts in the domain.
- If gap_type is "low_confidence", ask for clarification or confirmation of the existing relation.
- If gap_type is "unexplored", ask an opening question about this topic.
- Output ONLY the question text. No preamble, no quotes, no explanation.
"""


def generate_question(agent_type: str, agent_goal: str, task_boundary: str) -> QuestionGeneratorResult:
    """
    Run the gap detector, take the top gap, and generate one focused question.
    Returns None-equivalent (raises NoGapsRemaining) if the graph has no gaps left —
    this signals the caller (apprentice.py route) that the agent may be ready to propose.
    """
    gap_result: GapDetectorResult = detect_gaps(agent_type)

    if not gap_result.gaps:
        raise NoGapsRemaining(agent_type)

    top_gap = gap_result.gaps[0]

    known_concepts = _get_high_confidence_concepts(agent_type, exclude_id=top_gap.concept_id)

    prompt = QUESTION_PROMPT.format(
        goal=agent_goal,
        task_boundary=task_boundary,
        concept_label=top_gap.concept_label,
        gap_type=top_gap.gap_type,
        namespace=top_gap.namespace,
        known_concepts="\n".join(f"- {c}" for c in known_concepts) or "(none yet)",
    )

    response = _client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=200,
        messages=[{"role": "user", "content": prompt}],
    )
    question_text = response.content[0].text.strip()

    return QuestionGeneratorResult(
        agent_type_id=agent_type,
        question=question_text,
        targeting_gap=top_gap,
        is_followup=False,
    )


def generate_followup(
    agent_type: str,
    agent_goal: str,
    task_boundary: str,
    original_question: str,
    ambiguous_answer: str,
    targeting_gap: KnowledgeGap,
) -> QuestionGeneratorResult:
    """
    Called by T1.4's answer processor when status='follow_up'.
    Generates a clarifying question based on the ambiguous answer received.
    """
    prompt = f"""You asked: "{original_question}"
The expert answered: "{ambiguous_answer}"

This answer was too ambiguous to extract structured knowledge from. Ask ONE \
follow-up question that would clarify the ambiguity. Stay within this task boundary: \
{task_boundary}

Output ONLY the follow-up question text."""

    response = _client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=150,
        messages=[{"role": "user", "content": prompt}],
    )
    question_text = response.content[0].text.strip()

    return QuestionGeneratorResult(
        agent_type_id=agent_type,
        question=question_text,
        targeting_gap=targeting_gap,
        is_followup=True,
    )


def _get_high_confidence_concepts(agent_type: str, exclude_id: str, threshold: float = 0.75,
                                   limit: int = 20) -> list[str]:
    """
    Concepts with high-confidence edges, used for deduplication context.
    Capped at `limit` to keep the prompt small as the graph grows.
    """
    df = execute(agent_type, """
        MATCH (a:Concept)-[r:RELATES_TO]->(b:Concept)
        WHERE r.confidence >= $threshold AND a.id <> $exclude_id
        RETURN DISTINCT a.label AS label
        LIMIT $limit
    """, {"threshold": threshold, "exclude_id": exclude_id, "limit": limit}).get_as_df()

    return df["label"].tolist()


class NoGapsRemaining(Exception):
    def __init__(self, agent_type: str):
        self.agent_type = agent_type
        super().__init__(f"No knowledge gaps remaining for '{agent_type}' — graph may be saturated")
```

### API Route

```python
# src/api/routes/apprentice.py

from fastapi import APIRouter, HTTPException
from src.apprentice.question_generator import generate_question, NoGapsRemaining
from src.core.agent_type_service import get_agent_type_by_name
from src.core.enums import Phase

router = APIRouter(prefix="/agent-types", tags=["apprentice"])


@router.post("/{agent_type_id}/qa/question")
async def get_next_question(agent_type_id: str):
    agent = await get_agent_type_by_name(agent_type_id)  # or by id, per T1.1 convention

    if agent.phase != Phase.APPRENTICE:
        raise HTTPException(status_code=409, detail=f"Agent is in {agent.phase.value} phase, not apprentice")

    try:
        result = generate_question(agent.name, agent.goal, agent.task_boundary)
    except NoGapsRemaining:
        raise HTTPException(status_code=200, detail="no_gaps_remaining")  # see Notes

    return result
```

## Acceptance Criteria

1. `generate_question()` on a graph with at least one orphan node returns a question targeting that orphan's `concept_label`
2. The generated question never contains text that matches a concept in `_get_high_confidence_concepts()` output verbatim (deduplication check — approximate, via substring match in tests)
3. `generate_question()` on a graph with zero gaps raises `NoGapsRemaining`
4. `generate_followup()` produces a question different from the `original_question` passed in
5. The prompt always includes `task_boundary` — verified by inspecting the constructed prompt string in a test (mock the Anthropic client)
6. `POST /qa/question` on an agent not in apprentice phase returns `409`

def test_deduplication_excludes_high_confidence_concepts(agent_type):

## Notes

- **Model choice:** `claude-sonnet-4-6` is used for question generation — this is a
  lightweight, frequent call (one per Q&A turn) where Sonnet's latency/cost profile is
  appropriate. Reserve Opus-tier models for T1.7 (team proposal), which runs far less often
  and benefits more from deeper reasoning.
- **`NoGapsRemaining` as a signal, not an error:** when the gap detector returns an empty
  list, this usually means the readiness score is high and the agent should move toward
  proposal/cert rather than asking more questions. The API route returns this as a `200`
  with a `no_gaps_remaining` body rather than a `5xx` — it's an expected state transition,
  not a failure. The frontend (simulation console or Q&A UI) should interpret this as
  "check `/readiness` and consider triggering `/proposal`".
- **Deduplication is approximate by design.** `_get_high_confidence_concepts` provides
  context to the LLM so it *tends* not to re-ask, but this is a prompt-engineering soft
  constraint, not a hard filter. A hard filter would require semantic similarity matching
  on every candidate question, which is unnecessary complexity for this stage — the
  contradiction/ambiguity detection in T1.4 is the actual safety net for redundant or
  conflicting answers.
- **Why generate the question from the gap, not from raw graph state:** keeping the gap
  detector (T1.2) and question generator (T1.3) as separate stages means the ranking logic
  (severity × breadth) is testable independently of LLM behavior, and the same gap list
  could in principle drive a different question-asking strategy later without touching T1.2.
