## What to Build

Given a message sent from one simulated agent role to another, validates the message's
fields against the *receiving* agent's `input_schema` and `prohibited_fields` as defined
in the approved team composition. Returns a list of `ContractViolation` -- empty if the
message is fully compliant.

This module is called by T1.9 on every simulation turn, before the turn is recorded as
successful. It does not resolve violations itself -- that's a separate API call
(`POST /simulations/{id}/violations/{index}/resolve`) also implemented in T1.9.

```python
# src/apprentice/simulation/contract_validator.py

from src.core.models import AgentContract, ContractViolation


def validate_message(
    message_fields: dict,
    from_agent: str,
    to_agent: str,
    agents: list[AgentContract],
) -> list[ContractViolation]:
    """
    Validate `message_fields` (the structured payload of a simulation turn message)
    against `to_agent`'s declared input_schema and prohibited_fields.

    Two violation types:
      1. PROHIBITED FIELD -- message contains a field that to_agent's contract
         explicitly says it must never receive (e.g. card_token sent to a kitchen agent).
         This is always a violation regardless of input_schema.
      2. UNDECLARED FIELD -- message contains a field not in to_agent's input_schema
         at all. This may indicate the contract is incomplete (needs clarify_contract)
         or the sending agent is over-sharing (needs mark_edge_case).

    Returns an empty list if the message is fully compliant.
    """
    receiving_contract = _find_contract(to_agent, agents)
    if receiving_contract is None:
        return [
            ContractViolation(
                field=field, reason=f"'{to_agent}' has no contract in the approved composition",
                from_agent=from_agent, to_agent=to_agent,
            )
            for field in message_fields.keys()
        ]

    violations: list[ContractViolation] = []

    for field in message_fields.keys():
        if field in receiving_contract.prohibited_fields:
            violations.append(ContractViolation(
                field=field,
                reason=f"'{to_agent}' is prohibited from receiving '{field}'",
                from_agent=from_agent, to_agent=to_agent,
            ))
        elif field not in receiving_contract.input_schema:
            violations.append(ContractViolation(
                field=field,
                reason=f"'{field}' is not declared in '{to_agent}'s input_schema",
                from_agent=from_agent, to_agent=to_agent,
            ))

    return violations


def apply_resolution(
    violation: ContractViolation,
    resolution: str,
    clarification: str | None,
    agents: list[AgentContract],
) -> tuple[list[AgentContract], bool]:
    """
    Apply an expert's resolution to a violation.

    resolution == "clarify_contract":
        Add the violating field to the receiving agent's input_schema with the
        type/description given in `clarification`. Returns updated agents list
        and coord_edge_written=True (the caller, T1.9, will write a HANDS_OFF_TO
        edge update to Kuzu reflecting the schema change).

    resolution == "mark_edge_case":
        No schema change. Returns the agents list unchanged and
        coord_edge_written=False. The violation is logged as an observation
        but does not alter the contract -- this was a one-off, not a pattern.

    Returns (updated_agents, coord_edge_written).
    """
    if resolution == "mark_edge_case":
        return agents, False

    if resolution == "clarify_contract":
        if not clarification:
            raise ValueError("clarification is required when resolution='clarify_contract'")

        updated_agents = []
        for agent in agents:
            if agent.agent_name == violation.to_agent:
                new_input_schema = dict(agent.input_schema)
                new_input_schema[violation.field] = clarification
                updated_agents.append(agent.model_copy(update={"input_schema": new_input_schema}))
            else:
                updated_agents.append(agent)
        return updated_agents, True

    raise ValueError(f"Unknown resolution type: {resolution!r}")


def _find_contract(agent_name: str, agents: list[AgentContract]) -> AgentContract | None:
    return next((a for a in agents if a.agent_name == agent_name), None)
```

## Acceptance Criteria

1. A message containing only fields present in `to_agent.input_schema` (and none in
   `prohibited_fields`) returns an empty violations list
2. A message containing a field listed in `to_agent.prohibited_fields` returns a
   violation with that field, regardless of whether the field is also in `input_schema`
3. A message containing a field not in `input_schema` and not in `prohibited_fields`
   returns an "undeclared field" violation
4. `apply_resolution(..., resolution="clarify_contract", clarification="str")` adds the
   violating field to the matching agent's `input_schema` and returns
   `coord_edge_written=True`
5. `apply_resolution(..., resolution="mark_edge_case", ...)` returns the agents list
   unchanged and `coord_edge_written=False`
6. `apply_resolution(..., resolution="clarify_contract", clarification=None)` raises
   `ValueError`
7. Validating a message against a `to_agent` not present in `agents` returns one
   violation per field in the message, each citing the missing contract

```python
# tests/phase_1/test_contract_validator.py

import pytest
from src.apprentice.simulation.contract_validator import validate_message, apply_resolution
from src.core.models import AgentContract


@pytest.fixture
def agents():
    return [
        AgentContract(
            agent_name="order-agent",
            input_schema={"items": "list[str]"},
            output_schema={"order_id": "str", "items": "list[str]"},
            prohibited_fields=[],
        ),
        AgentContract(
            agent_name="kitchen-agent",
            input_schema={"order_id": "str", "items": "list[str]"},
            output_schema={"est_minutes": "int"},
            prohibited_fields=["card_token", "customer_address"],
        ),
    ]


def test_compliant_message_no_violations(agents):
    violations = validate_message(
        message_fields={"order_id": "ord-1", "items": ["burger"]},
        from_agent="order-agent", to_agent="kitchen-agent", agents=agents,
    )
    assert violations == []


def test_prohibited_field_detected(agents):
    violations = validate_message(
        message_fields={"order_id": "ord-1", "items": ["burger"], "card_token": "tok_abc"},
        from_agent="order-agent", to_agent="kitchen-agent", agents=agents,
    )
    assert len(violations) == 1
    assert violations[0].field == "card_token"
    assert "prohibited" in violations[0].reason


def test_undeclared_field_detected(agents):
    violations = validate_message(
        message_fields={"order_id": "ord-1", "items": ["burger"], "table_number": 7},
        from_agent="order-agent", to_agent="kitchen-agent", agents=agents,
    )
    assert len(violations) == 1
    assert violations[0].field == "table_number"
    assert "not declared" in violations[0].reason


def test_prohibited_takes_precedence_even_if_also_in_schema(agents):
    agents[1].input_schema["card_token"] = "str"
    violations = validate_message(
        message_fields={"order_id": "ord-1", "items": ["burger"], "card_token": "tok_abc"},
        from_agent="order-agent", to_agent="kitchen-agent", agents=agents,
    )
    assert len(violations) == 1
    assert violations[0].field == "card_token"
    assert "prohibited" in violations[0].reason


def test_missing_contract_returns_violation_per_field(agents):
    violations = validate_message(
        message_fields={"a": 1, "b": 2},
        from_agent="order-agent", to_agent="nonexistent-agent", agents=agents,
    )
    assert len(violations) == 2
    assert all("no contract" in v.reason for v in violations)


def test_clarify_contract_adds_field_to_schema(agents):
    from src.core.models import ContractViolation
    violation = ContractViolation(
        field="table_number", reason="not declared",
        from_agent="order-agent", to_agent="kitchen-agent",
    )
    updated_agents, coord_written = apply_resolution(
        violation, resolution="clarify_contract", clarification="int", agents=agents,
    )
    kitchen = next(a for a in updated_agents if a.agent_name == "kitchen-agent")
    assert kitchen.input_schema["table_number"] == "int"
    assert coord_written is True

    original_kitchen = next(a for a in agents if a.agent_name == "kitchen-agent")
    assert "table_number" not in original_kitchen.input_schema


def test_mark_edge_case_no_schema_change(agents):
    from src.core.models import ContractViolation
    violation = ContractViolation(
        field="table_number", reason="not declared",
        from_agent="order-agent", to_agent="kitchen-agent",
    )
    updated_agents, coord_written = apply_resolution(
        violation, resolution="mark_edge_case", clarification=None, agents=agents,
    )
    assert updated_agents == agents
    assert coord_written is False


def test_clarify_contract_without_clarification_raises(agents):
    from src.core.models import ContractViolation
    violation = ContractViolation(
        field="table_number", reason="not declared",
        from_agent="order-agent", to_agent="kitchen-agent",
    )
    with pytest.raises(ValueError):
        apply_resolution(violation, resolution="clarify_contract", clarification=None, agents=agents)
```

## Notes

- **Prohibited-field check runs before the input_schema check**, and a field can in
  principle appear in both `input_schema` and `prohibited_fields` (a contradictory
  contract an expert wrote during T1.8's edit flow). The prohibited check wins -- this
  reflects that `prohibited_fields` represents a hard safety boundary (the same boundary
  T3.1's action manifest enforces in Phase 3), while `input_schema` is a structural/typing
  declaration. A field being prohibited is a stronger statement than it being typed.
- **`apply_resolution` never mutates the input `agents` list** -- it uses
  `model_copy(update=...)` to produce new `AgentContract` instances. This matters because
  T1.9's simulation session holds the in-memory `agents` list for the duration of a
  session and must be able to compare before/after states when deciding whether to write
  a `HANDS_OFF_TO` edge update to Kuzu at session close.
- **`mark_edge_case` deliberately does nothing to the schema.** Its only effect (handled
  by T1.9, not this module) is recording a `SimulationObservation` with
  `type=ObservationType.VIOLATION` and `coord_edge_written=False`. This is correct for
  genuinely one-off scenarios -- e.g. "the expert played a role slightly out of character
  for this one turn" -- that shouldn't permanently widen a contract.
- **This module has no knowledge of Kuzu, Postgres, or the simulation session state.**
  It is pure functions over in-memory data structures, which is why its test suite needs
  no fixtures beyond a plain `agents` list. T1.9 is responsible for all persistence
  side-effects (writing `HANDS_OFF_TO` edges, updating `team_compositions`, recording
  `SimulationObservation`s).
