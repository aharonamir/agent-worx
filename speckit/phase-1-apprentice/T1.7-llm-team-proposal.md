# T1.7 — LLM Team Proposal

## Dependencies
- T1.6 complete (`compute_and_store()` returns `ReadinessScore` with `proposal_ready` flag)
- T0.7 complete (`team_compositions` table exists)
- Load `shared/DATA_SCHEMAS.md` — produces `ProposalArtifact`, `AgentContract`, `HandoffContract`
- Anthropic API client configured

## Files to Create
- `src/apprentice/proposal_engine.py`

## What to Build

Triggered when `readiness_score.proposal_ready == True` (score ≥ 0.55). Clusters the domain
graph into candidate agents, identifies handoff points between clusters, and produces a
`ProposalArtifact` with full rationale. For a single-domain agent (no natural clusters),
this produces a single-agent proposal.

### Algorithm Overview

1. **Cluster the graph.** Run connected-component analysis on the Kuzu domain graph
   (task namespace only). Each component with at least `MIN_CLUSTER_SIZE` nodes and a
   coherent theme is a candidate agent.
2. **Identify cross-cluster edges.** Edges connecting nodes in different clusters are
   candidate handoff points.
3. **LLM synthesis.** Pass the cluster summaries plus cross-cluster edges to the LLM,
   which produces agent names, roles, I/O contracts, and rationale for each agent and
   contract.
4. **Persist.** Write the resulting `ProposalArtifact` to `team_compositions` with
   `status="proposed"` and `version = (max existing version for this agent_type) + 1`.

```python
# src/apprentice/proposal_engine.py

import json
import uuid
from datetime import datetime
from anthropic import Anthropic
from src.core.models import ProposalArtifact, AgentContract, HandoffContract
from src.core.enums import TopologyType, CompositionStatus
from src.infra.kuzu_client import execute
from src.infra.postgres_client import get_pool

_client = Anthropic()

MIN_CLUSTER_SIZE = 8

PROPOSAL_PROMPT_TEMPLATE = (
    "You are designing the team structure for a multi-agent system based on a "
    "knowledge graph built through Socratic Q&A with a domain expert.\n\n"
    "AGENT DOMAIN:\n"
    "  Goal: {goal}\n"
    "  Task boundary: {task_boundary}\n"
    "  Declared topology preference: {topology_type}\n"
    "  Declared workflow participants: {workflow_participants}\n\n"
    "GRAPH CLUSTERS (each is a candidate agent's area of responsibility):\n"
    "{cluster_summaries}\n\n"
    "CROSS-CLUSTER RELATIONSHIPS (candidate handoff points between agents):\n"
    "{cross_cluster_edges}\n\n"
    "YOUR TASK:\n"
    "Propose a team composition. For EACH agent provide:\n"
    "  - agent_name (kebab-case)\n"
    "  - input_schema (field name to type string)\n"
    "  - output_schema (field name to type string)\n"
    "  - prohibited_fields (data this agent must never receive)\n\n"
    "For EACH handoff provide:\n"
    "  - from_agent, to_agent\n"
    "  - condition (when this handoff occurs)\n"
    "  - validates (fields confirmed before handoff)\n"
    "  - rationale\n\n"
    "If only one coherent cluster exists, propose a single agent with "
    "topology=pipeline and an empty contracts list.\n\n"
    "Respond ONLY with JSON, no markdown fences, in this exact shape:\n"
    "{{\n"
    '  "topology": "pipeline|orchestrated|peer",\n'
    '  "agents": [{{"agent_name": "...", "input_schema": {{}}, "output_schema": {{}}, "prohibited_fields": []}}],\n'
    '  "contracts": [{{"from_agent": "...", "to_agent": "...", "condition": "...", "validates": [], "rationale": "..."}}],\n'
    '  "rationale_per_agent": {{"agent_name": "..."}},\n'
    '  "rationale_per_contract": {{"from->to": "..."}}\n'
    "}}\n"
)


def generate_proposal(agent_type: str, agent_goal: str, task_boundary: str,
                       declared_topology: TopologyType,
                       workflow_participants: list[str]) -> ProposalArtifact:
    """
    Generate a team proposal from the current domain graph state.
    Persists to team_compositions with status='proposed'.
    """
    clusters = _find_clusters(agent_type)
    cross_edges = _find_cross_cluster_edges(agent_type, clusters)

    prompt = PROPOSAL_PROMPT_TEMPLATE.format(
        goal=agent_goal,
        task_boundary=task_boundary,
        topology_type=declared_topology.value,
        workflow_participants=", ".join(workflow_participants) or "(none declared)",
        cluster_summaries=_format_cluster_summaries(clusters),
        cross_cluster_edges=_format_cross_cluster_edges(cross_edges),
    )

    response = _client.messages.create(
        model="claude-opus-4-6",
        max_tokens=3000,
        messages=[{"role": "user", "content": prompt}],
    )

    text = response.content[0].text.strip()
    if text.startswith("```"):
        text = text.split("```")[1]
        if text.startswith("json"):
            text = text[4:]
    parsed = json.loads(text.strip())

    artifact = ProposalArtifact(
        agent_type_id=agent_type,
        version=1,
        topology=TopologyType(parsed["topology"]),
        agents=[AgentContract(**a) for a in parsed["agents"]],
        contracts=[HandoffContract(**c) for c in parsed["contracts"]],
        rationale_per_agent=parsed["rationale_per_agent"],
        rationale_per_contract=parsed["rationale_per_contract"],
        status=CompositionStatus.PROPOSED,
    )

    return _persist_proposal(agent_type, artifact)


async def _persist_proposal(agent_type: str, artifact: ProposalArtifact) -> ProposalArtifact:
    pool = get_pool()
    async with pool.acquire() as conn:
        max_version_row = await conn.fetchrow(
            "SELECT MAX(version) AS max_v FROM team_compositions WHERE agent_type = $1", agent_type
        )
        next_version = (max_version_row["max_v"] or 0) + 1
        artifact.version = next_version

        await conn.execute(
            "INSERT INTO team_compositions "
            "(id, agent_type, version, topology, agent_list, contracts, rationale, status, created_at) "
            "VALUES ($1, $2, $3, $4, $5, $6, $7, $8, $9)",
            artifact.id, agent_type, next_version, artifact.topology.value,
            json.dumps([a.model_dump() for a in artifact.agents]),
            json.dumps([c.model_dump() for c in artifact.contracts]),
            json.dumps({
                "rationale_per_agent": artifact.rationale_per_agent,
                "rationale_per_contract": artifact.rationale_per_contract,
            }),
            CompositionStatus.PROPOSED.value, artifact.created_at,
        )
    return artifact


# Graph clustering --------------------------------------------------------------

def _find_clusters(agent_type: str) -> list[dict]:
    """
    Connected-component analysis on the task namespace via union-find.
    Components with at least MIN_CLUSTER_SIZE nodes become separate clusters;
    smaller components are merged into one "general" cluster.
    """
    nodes_df = execute(agent_type,
        "MATCH (c:Concept) WHERE c.namespace = 'task' RETURN c.id AS id, c.label AS label"
    ).get_as_df()
    edges_df = execute(agent_type,
        "MATCH (a:Concept)-[:RELATES_TO]-(b:Concept) "
        "WHERE a.namespace = 'task' AND b.namespace = 'task' "
        "RETURN a.id AS source, b.id AS target"
    ).get_as_df()

    parent = {row["id"]: row["id"] for _, row in nodes_df.iterrows()}

    def find(x):
        while parent[x] != x:
            x = parent[x]
        return x

    def union(a, b):
        ra, rb = find(a), find(b)
        if ra != rb:
            parent[ra] = rb

    for _, row in edges_df.iterrows():
        if row["source"] in parent and row["target"] in parent:
            union(row["source"], row["target"])

    groups: dict[str, list[str]] = {}
    label_lookup = dict(zip(nodes_df["id"], nodes_df["label"]))
    for node_id in parent:
        root = find(node_id)
        groups.setdefault(root, []).append(node_id)

    clusters = []
    general_labels = []
    for root, member_ids in groups.items():
        labels = [label_lookup[m] for m in member_ids]
        if len(member_ids) >= MIN_CLUSTER_SIZE:
            clusters.append({"id": root, "node_ids": member_ids, "labels": labels})
        else:
            general_labels.extend(labels)

    if general_labels:
        clusters.append({"id": "general", "node_ids": [], "labels": general_labels})

    return clusters


def _find_cross_cluster_edges(agent_type: str, clusters: list[dict]) -> list[dict]:
    node_to_cluster: dict[str, str] = {}
    for cluster in clusters:
        for node_id in cluster["node_ids"]:
            node_to_cluster[node_id] = cluster["id"]

    edges_df = execute(agent_type,
        "MATCH (a:Concept)-[r:RELATES_TO]->(b:Concept) "
        "WHERE a.namespace = 'task' AND b.namespace = 'task' "
        "RETURN a.id AS source_id, a.label AS source_label, "
        "b.id AS target_id, b.label AS target_label, r.relation AS relation"
    ).get_as_df()

    cross_edges = []
    for _, row in edges_df.iterrows():
        sc = node_to_cluster.get(row["source_id"])
        tc = node_to_cluster.get(row["target_id"])
        if sc and tc and sc != tc:
            cross_edges.append({
                "source_label": row["source_label"], "source_cluster": sc,
                "target_label": row["target_label"], "target_cluster": tc,
                "relation": row["relation"],
            })
    return cross_edges


def _format_cluster_summaries(clusters: list[dict]) -> str:
    lines = []
    for i, cluster in enumerate(clusters):
        sample = ", ".join(cluster["labels"][:15])
        more = " ..." if len(cluster["labels"]) > 15 else ""
        lines.append(f"Cluster {i+1} ({cluster['id']}, {len(cluster['labels'])} concepts): {sample}{more}")
    return "\n".join(lines) if lines else "(single undifferentiated cluster)"


def _format_cross_cluster_edges(edges: list[dict]) -> str:
    if not edges:
        return "(none -- single cluster, propose a single agent)"
    lines = []
    for e in edges[:30]:
        lines.append(f"- {e['source_label']} ({e['source_cluster']}) "
                      f"--[{e['relation']}]--> {e['target_label']} ({e['target_cluster']})")
    return "\n".join(lines)
```

## Acceptance Criteria

1. A graph with a single connected component produces a `ProposalArtifact` with exactly
   one agent and `topology=TopologyType.PIPELINE`
2. A graph with two clearly separable clusters connected by one cross-edge produces a
   proposal with at least 2 agents and at least 1 contract connecting them
3. Every agent in `artifact.agents` has a corresponding entry in `rationale_per_agent`
4. Every contract in `artifact.contracts` has a corresponding entry in
   `rationale_per_contract`, keyed as `"{from_agent}->{to_agent}"`
5. `_persist_proposal` assigns `version=1` for the first proposal of an agent type, and
   `version=N+1` if N proposals already exist regardless of their status
6. The persisted row in `team_compositions` has `status='proposed'`

## Notes

- **Model choice: `claude-opus-4-6`, not Sonnet.** Unlike T1.3's per-turn question
  generation, the team proposal runs once (or a handful of times, on rejection or
  re-proposal) per agent type's entire lifecycle. The reasoning required for clustering
  interpretation, contract design, and rationale benefits from the deeper model, and the
  cost is negligible at this call frequency.
- **Union-find clustering runs in application code, not in Kuzu.** The pinned Kuzu version
  in `shared/DEPENDENCIES.md` does not ship a connected-components algorithm. Fetching all
  task-namespace nodes and edges into a DataFrame and running union-find in Python is
  O(V+E) and entirely sufficient at apprentice-phase graph sizes (typically hundreds of
  nodes). If a graph grows into the tens of thousands of nodes before reaching
  `proposal_ready`, that is itself a signal the domain seed (T1.1) was too broad.
- **The "general" cluster catch-all** ensures small disconnected fragments don't each spawn
  their own agent proposal. They are folded into one bucket presented to the LLM as
  miscellaneous context, and typically get assigned to whichever proposed agent fits best.
- **`prohibited_fields` becomes the action manifest's `prohibited_fields`** enforced in
  Phase 3 (T3.1) and validated at runtime by the contract validator during simulation
  (T1.10). Getting this right at proposal time — e.g. the order-agent example explicitly
  prohibiting `card_token` — is exactly what T1.10's simulation violations check against.
- **`_persist_proposal` is shown as `async` but called from a sync function
  (`generate_proposal`).** In the route handler (T1.8), `generate_proposal` should be
  `async def` end-to-end; either run the Anthropic call in a thread executor or switch to
  `AsyncAnthropic`. Apply the same approach consistently across T1.3, T1.4, and T1.7.
